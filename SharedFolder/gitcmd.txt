===============================
Git 常用命令速查表
===============================

【查看状态】
git status                          # 查看当前状态
git log                             # 查看提交历史
git log --oneline                   # 简洁查看历史
git log --graph --oneline --all     # 图形化查看所有分支

【分支操作】
git branch                          # 列出本地分支
git branch -a                       # 列出所有分支（包括远程）
git checkout main                   # 切换到 main 分支
git checkout cpp-modernization      # 切换到 cpp-modernization 分支
git checkout -b 新分支名             # 创建并切换到新分支
git branch -d 分支名                 # 删除本地分支
git branch -D 分支名                 # 强制删除本地分支

【提交流程】
git add .                           # 添加所有修改到暂存区
git add 文件名                       # 添加指定文件
git commit -m "提交信息"             # 提交到本地仓库
git push                            # 推送到远程仓库
git push origin cpp-modernization   # 推送指定分支到远程

【撤销操作】
git checkout -- .                   # 丢弃所有未提交的修改
git checkout -- 文件名               # 丢弃指定文件的修改
git restore 文件名                   # 撤销对文件的修改（新版命令）
git restore .                       # 撤销所有修改
git reset HEAD~1                    # 撤销最后一次提交（保留修改）
git reset --hard HEAD~1             # 撤销最后一次提交（删除修改，危险！）

【暂存操作】
git stash                           # 暂存当前修改
git stash push -m "说明"             # 暂存并添加说明
git stash list                      # 查看所有暂存
git stash pop                       # 恢复最近的暂存并删除
git stash apply                     # 恢复最近的暂存但不删除
git stash drop                      # 删除最近的暂存

【对比差异】
git diff                            # 查看未暂存的修改
git diff --staged                   # 查看已暂存的修改
git diff main cpp-modernization     # 对比两个分支

【合并分支】
git checkout main                   # 先切到目标分支
git merge cpp-modernization         # 合并 cpp-modernization 到 main

【远程操作】
git pull                            # 拉取远程更新
git pull origin main                # 拉取指定分支
git push origin main                # 推送到远程 main 分支
git push origin --delete 分支名      # 删除远程分支

【查看远程】
git remote -v                       # 查看远程仓库地址
git remote show origin              # 查看远程仓库详细信息

===============================
当前项目常用操作
===============================

【切换到 C++ 版本】
git checkout cpp-modernization

【切回 C 版本】
git checkout main

【丢弃编译生成的临时文件】
git checkout -- .

【提交当前修改】
git add .
git commit -m "修改说明"
git push

【如果 C++ 版本测试通过，合并到 main】
git checkout main
git merge cpp-modernization
git push

【推送 C++ 分支到 GitHub】
git push origin cpp-modernization

===============================
注意事项
===============================

1. 编译生成的文件（.o, .axf, .hex 等）不需要提交
2. Keil 项目配置文件（.uvguix, .uvoptx）一般也不提交
3. 切换分支前记得保存或提交修改
4. 使用 git status 随时查看当前状态
5. 不确定时先用 git stash 保存修改
